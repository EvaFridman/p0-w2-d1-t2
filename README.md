# p0-w2-d1-t2 · Резиновая панель

> Тип: новый материал · Репозиторий: один отдельный

## Задача

Верхняя панель сайта: слева логотип, в центре строка поиска, справа кнопка входа. Логотип и
кнопка занимают по содержимому, а поиск растягивается на всё свободное место. Когда окно
сужается — поиск ужимается первым, а логотип и кнопка не ломаются. Это делается свойствами
самих флекс-элементов: `flex-grow`, `flex-shrink`, `flex-basis`.

```
панель: [ лого ][  поиск (тянется)  ][ вход ]
          basis     flex-grow:1       basis

окно шире  → растягивается только поиск
окно уже   → flex-shrink ужимает, не ломая
```

## Что нужно сделать

Свёрстать такую панель и настроить поведение элементов при изменении ширины окна. Поиск —
тянущийся, боковые элементы — по содержимому. Проверять, перетаскивая край окна или меняя
ширину в DevTools.

## Что нужно получить

- логотип и кнопка входа занимают столько, сколько нужно их содержимому;
- поле поиска забирает всё оставшееся место (`flex-grow`);
- при сужении окна уменьшается поиск, а логотип и кнопка держат размер (подумай, как запретить
  сжатие именно боковым элементам — по умолчанию ужимаются все);
- ничего не вылезает за пределы панели и не наезжает друг на друга.

## Критерии приёмки

* Раскладка держится на свойствах элементов (`flex-grow`/`shrink`/`basis` или сокращение
  `flex`), а не на фиксированных ширинах в пикселях.
* При изменении ширины окна панель ведёт себя как описано: тянется и ужимается поиск.
* В `CSS.md` своими словами: что делают `flex-grow`, `flex-shrink`, `flex-basis` и что значит
  запись `flex: 1 1 auto`.
